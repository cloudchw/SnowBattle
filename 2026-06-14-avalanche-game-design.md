## 1. 产品定位

### 1.1 基本信息

| 维度 | 内容 |
|---|---|
| **产品名（暂定）** | 《雪炮突击队》（Avalanche Squad） |
| **一句话定位** | 一款"用大炮主动制造雪崩以保护滑雪者"的策略射击小游戏 |
| **目标平台** | 微信小游戏 + 抖音小游戏（首发） |
| **核心情绪** | 紧张 + 爽快 + 略带黑色幽默（俄罗斯大叔 + 雪崩喜剧） |
| **目标用户** | 微信生态内 18–40 岁休闲玩家；喜欢《愤怒的小鸟》《割绳子》类策略小品 |
| **典型场景** | 通勤/排队/午休的单次 1–3 分钟碎片时间 |
| **差异化卖点** | "我是引发雪崩的人，但雪崩会反噬我" — 反向自保机制是该品类几乎没人做的 |

### 1.2 故事背景

> 你是俄罗斯高加索山区一名经验丰富的雪炮作业员。每个冬天清晨，游客们迫不及待要冲进雪道，但山坡上积着不稳定雪层 — 一声咳嗽就可能引发致命雪崩。你的任务：在他们冲下来之前，用各种火炮精准轰掉危险雪层，让山坡"安全卸货"。但记住了 — 雪崩不分敌我，轰得太狠、太近、方向错了，第一个被埋的就是你自己。

### 1.3 核心循环

**单关循环（60–90 秒）**

```
开局 → 旋转/缩放观察雪山 → 识别红色危险雪区 + 蓝色游客路径
     → 倒计时启动（游客 30/60/90 秒后入场）
     → 选炮弹 + 拖拽瞄准 + 松手发射（仅方向+力度反馈，无弹道预览）
     → 命中：雪层剥落 → 形成雪崩 → 雪崩沿地形流下
     → 三种结果：
         ✅ 危险雪层清除完毕 + 游客安全通过 = 过关（按用时/炮弹数评 1-3 星）
         ❌ 倒计时归零仍有未清危险雪区 = 失败
         ❌ 雪崩流到炮位/游客区 = 失败（喜剧式被埋动画）
```

**跨关循环**

```
通关 → 获得金币 + 经验 → 升级炮台/解锁新炮弹 → 挑战下一关
失败 → 激励视频复活一次（保留进度）/ 重试
```

### 1.4 三大核心张力

设计之魂 — 每个机制都应至少强化其中之一：

1. **观察 vs 时间**：倒计时一直在跑，但你必须先看懂地形才能开炮 — 急躁会送命
2. **威力 vs 自保**：大威力的雪崩能一次清干净，但也可能流到自己脚下 — 贪心会送命
3. **范围 vs 误伤**：集束弹覆盖广，但游客已经进入雪道时你不敢用 — 时机选择是核心策略

## 2. 核心玩法机制

### 2.1 战场布局（2.5D 等距视角）

### 2.2 五种炮弹系统（核心策略维度）

| # | 名称 | 视觉 | 机制 | 典型场景 | 抗风性 | 弹药限制 |
|---|---|---|---|---|---|---|
| 1 | **标准高爆弹** | 红色圆润 | 中等伤害 + 中等范围 + 中等震动 | 大部分不稳定雪层 | 中 | 关卡默认 3-5 发 |
| 2 | **重型榴弹** | 黑色粗壮 | 高伤害 + 小范围 + 大震动 | 坚冰层、深层临界雪 | 强 | 1-2 发 |
| 3 | **集束弹** | 黄色分裂 | 空中分裂成 5 枚覆盖广域 + 低震动 | 大片散布的不稳定雪层 | 弱 | 1-2 发 |
| 4 | **热熔弹** | 橙色火焰 | 范围加热融化雪层，**不引发雪崩** | 游客已经入场，仍需清的小块危险 | 中 | 1 发（救命弹） |
| 5 | **穿甲弹** | 蓝色尖锐 | 钻入深处引爆，引发**深层断裂雪崩** | 需要触发大规模定向雪崩清空整面坡 | 强 | 1 发（核武器） |

**设计要点**：每种炮弹解决一类问题，玩家必须读懂地形 + 雪层状态 + 游客时机，才能选对炮弹。误用炮弹（如游客在场时用了集束弹/穿甲弹）= 立刻失败。

### 2.3 雪层状态系统

雪层有 3 种可视化状态（玩家"观察"的核心信息）：

| 状态 | 颜色 | 含义 | 玩家应对 |
|---|---|---|---|
| 🟢 **稳定层** | 白色 | 安全，无需处理 | 不打 |
| 🟡 **不稳定层** | 黄色脉动 | 受到震动会松动 | 1 发标准弹可清 |
| 🔴 **临界层** | 红色强烈脉动 | 随时可能自然崩塌，倒计时归零自动崩 | 必须主动清掉，重炮/穿甲处理 |

**隐藏信息（高阶钻研点）**：
- **坚冰层**（蓝色半透明）：重炮才能碎，标准弹打上去只响一声
- **深积雪**（带阴影）：集束弹效率低需多打
- **空腔层**（雪下有缝隙，需观察细微裂纹）：穿甲弹能引发连锁雪崩

### 2.4 雪崩物理（自研轻量模拟）

**触发条件**：
- 临界层被命中
- 不稳定层被强力震动（重炮/穿甲爆炸）波及
- 坚冰被击碎波及下方

**流向规则**：
- 雪崩沿**地形等高线 + 坡度**自动流下（用粒子流模拟，非真物理引擎）
- 雪崩走到平地会**堆积**，玩家可利用堆积挡住后续雪崩（高级技巧）
- 雪崩波及炮位 = 失败；波及游客 = 失败；波及终点旗帜 = 失败

**连环雪崩机制**（Boss 关核心）：
- 上一发雪崩的堆积改变地形高度图
- 下一发弹道因此偏移，需要预判

### 2.5 NPC 游客系统

| 维度 | 设计 |
|---|---|
| **数量** | 3–8 人/关，随关卡难度递增 |
| **入场** | 倒计时归零时从起点排成纵队进入雪道 |
| **速度** | 可被玩家观察但不可被控制；玩家应在他们入场前完成清场 |
| **路径** | 关卡设计师预设的曲线，**关卡开始时高亮显示**给玩家 |
| **悲剧设计** | 游客入场后仍需清小块危险 → 只能用热熔弹（不引发雪崩）→ 这是热熔弹的存在意义 |

### 2.6 拖拽瞄准操作（核心 UX）

| 动作 | 效果 |
|---|---|
| 在战场任意位置按下拖动 | 显示从炮位出发的**方向箭头** |
| 拖动方向 | 决定射击方向（从炮位反向延伸） |
| 拖动长度 | 决定力度，HUD 显示**力度条**（彩色填充条，绿→黄→红 表示弱→中→强） |
| 松手 | 按当前选中的炮弹类型发射，**弹道完全黑盒** |
| 双指捏合 | 缩放战场（用于精确瞄准远处） |
| 双指旋转 | 旋转视角（2.5D 允许 ±45° 旋转） |

**关键设计决策：不提供弹道/落点预览。** 玩家凭方向箭头 + 力度条 + 经验判断，提升技能型挑战深度。

**环境系统（全关通用）**：
- 🚩 **风向旗**：飘扬在战场侧边，旗帜飘向 = 风去向；旗帜长度/弯曲度 = 风力大小（视觉化）
- 📊 **风向/风力 HUD**：顶部小窗显示精确数据，如"↘ 3级"（数字精确，便于策略计算）
- 弹道受风力影响：横风偏移、逆风减程、顺风增程
- 不同炮弹抗风性不同（见 2.2）

**新手过渡（第 1-3 关）**：
- 近距（屏幕一半内）、无风、地形平坦
- 让玩家形成"方向+力度→落点"的基础肌肉记忆

### 2.7 计分与失败判定

**过关条件（同时满足）**：
1. 清除所有 🔴 临界层
2. 清除 ≥80% 的 🟡 不稳定层
3. 倒计时内完成
4. 没有雪崩波及炮位/游客/终点

**星级评定**：

| 星级 | 用时 | 弹药用量 |
|---|---|---|
| ⭐ | ≤ 倒计时 | ≤ 总弹药 70% |
| ⭐⭐ | ≤ 倒计时 70% | ≤ 总弹药 50% |
| ⭐⭐⭐ | ≤ 倒计时 50% | ≤ 总弹药 30% |

追求 3 星 = 高度策略性，是核心玩家复玩动力。

**失败场景**（带喜剧动画）：
- 游客被埋 → "游客：这滑雪票真不值" + 重试/复活
- 自己被埋 → "炮手：我太贪了" + 重试/复活
- 倒计时归零 → "游客：我们上了！" + 重试

## 3. 关卡设计与难度曲线

### 3.1 关卡总量与章节划分

主线 **50 关**，预计玩家 7-14 天通关（休闲节奏）。**周更关卡**从第 4 周起每周追加 3 关，无限延展。

| 章节 | 关卡范围 | 主题 | 新机制引入 |
|---|---|---|---|
| **第 1 章：新人村** | 1-3 | 近距练习 | 基础操作（方向+力度），无游客无风 |
| | 4-6 | 风的初体验 | 风向旗 + 风力，仍无游客 |
| **第 2 章：高山** | 7-12 | 雪层识别 | 不稳定层（黄）+ 临界层（红） |
| | 13-15 | 多目标 | 同时清多片危险区 |
| **第 3 章：深雪** | 16-20 | 雪崩流向 | 利用堆积、避免回旋 |
| | 21-23 | 坚冰登场 | 重炮机制 |
| **第 4 章：风暴** | 24-28 | 强风/暴雪 | 4-5 级风、能见度下降 |
| | 29-30 | 集束弹专场 | 多目标散布 |
| **第 5 章：核武器** | 31-35 | 穿甲弹 | 深层断裂 |
| | 36-38 | 热熔弹专场 | 游客在场时的精准清除 |
| **第 6 章：终极考验** | 39-45 | 综合应用 | 全机制混合 |
| | 46-50 | Boss 关 | 暴风雪 + 大型雪山 + 8 游客 + 5 种危险区 |

### 3.3 难度曲线

| 关卡段 | 平均游客数 | 倒计时 | 危险区数 | 风力 | 弹药紧张度 |
|---|---|---|---|---|---|
| 1-3 | 0 | ∞ | 1-2 | 无 | 宽松 |
| 4-6 | 0 | ∞ | 2-3 | 1-2 级 | 宽松 |
| 7-12 | 2-3 | 60s | 2-3 | 2 级 | 适中 |
| 13-23 | 3-5 | 50s | 3-5 | 2-3 级 | 紧张 |
| 24-38 | 4-6 | 40s | 4-6 | 3-4 级 | 紧张 |
| 39-50 | 5-8 | 30-40s | 5-8 | 4-5 级 | 极紧张 |

**关键设计**：弹药紧张度逐步上升 — 早期"打偏了还有备用"，后期"每一发都必须命中"，强化策略感和压力感。

**第 1-6 关无游客的设计意图**：刻意分离"弹道+风"训练与"时间压力"训练。1-3 关专注方向+力度肌肉记忆；4-6 关叠加风向影响；从第 7 关起才叠加游客入场倒计时，避免新手同时面对三个变量。这是教学排序逻辑，并非游客系统非 MVP 必需 — 游客系统从第 7 关起是核心循环不可分割的一部分。

### 3.4 Boss 关设计（46-50 关）

- **第 46 关：双子暴风雪山** — 两座雪山需分别清理，弹药极度紧张
- **第 47 关：夜战** — 能见度仅 30%，依赖风向旗和听觉（炮口闪光照明）
- **第 48 关：游客乱入** — 8 名游客从不同路径入场，需规划清除顺序避免误伤
- **第 49 关：连环雪崩** — 上一发雪崩的堆积改变下一发的弹道，需要预判
- **第 50 关：风暴巅峰** — 5 级风 + 暴雪 + 大型雪山 + 8 游客 + 8 危险区 + 5 种炮弹各 1 发

### 3.5 周更关卡设计原则

- 每周 3 关，主题化（如"俄罗斯冬季节"、"元宵特辑"）
- 难度对标主线第 30-40 关（中高难度，给老玩家挑战）
- 含 1 关"创意关"：引入限时机制（如限时 15s 速攻、限弹 1 发极限挑战）

## 4. 美术与技术架构

### 4.1 美术 — 角色设计

**主角（炮手）"鲍里斯"**：
- 形象：圆胖、戴雷锋帽、八字胡、酒红鼻头、米色军大衣
- 表情库：观察（眯眼）/ 瞄准（伸舌头）/ 命中（得意）/ 被埋（仅一只手伸出雪堆挥舞）
- 阵营色：红色围巾

**NPC 游客（5 种造型）**：
1. 戴墨镜的酷滑手（红夹克）
2. 戴粉蝴蝶结的女孩（粉外套）
3. 圆滚滚的小孩（黄色连体雪服）
4. 老爷爷（米色毛衣 + 拐杖滑雪板）
5. 双板教练（深蓝制服 + 哨子）

每个游客被埋时有专属的滑稽动画（如蝴蝶结女孩："啊我的发型！"）

### 4.2 美术 — 场景元素

| 元素 | 设计要点 |
|---|---|
| **雪层状态视觉** | 🟢白色平稳 / 🟡黄色脉动+裂纹 / 🔴红色强烈脉动+碎石下落动画 |
| **坚冰层** | 蓝色半透明，敲击时有裂纹动画 |
| **风向旗** | 三角旗，按风力弯曲度 + 抖动幅度差异化 |
| **炮台** | 5 种炮管外观：红圆（标准）/ 黑粗（重炮）/ 黄蜂腰（集束）/ 橙喷火（热熔）/ 蓝尖（穿甲） |
| **爆炸特效** | 标准=黄白闪光+雪花喷溅 / 重炮=深红冲击波 / 集束=多点闪光 / 热熔=橙色熔化涟漪 / 穿甲=蓝色脉冲 |
| **雪崩** | 粒子流，沿地形等高线流动，前缘有"白色浪头"视觉 |

### 4.6 技术架构 — 雪崩物理方案

**关键决策：自研轻量雪崩模拟，不用真物理引擎。**

| 维度 | 真物理（Cannon.js/Bullet） | 自研轻量 | 决策 |
|---|---|---|---|
| 表现可控 | 难调，物理行为不可预测 | 流向可艺术化 | 自研胜 |
| 性能 | 大量刚体昂贵 | 粒子系统 + 高度图便宜 | 自研胜 |
| 开发量 | 接入快，调参慢 | 2-3 周自研 | 真物理快但后续痛苦 |
| 雪堆积 | 难做 | 高度图天然支持 | 自研胜 |

**自研方案核心**：
- 雪崩 = **粒子流系统**（~500 粒子/关），每个粒子有质量、速度、与地形交互规则
- 地形 = **高度图网格**（Grid 32×32 或 64×64），雪崩粒子根据高度差决定流向
- 雪堆积 = 粒子停止时叠加到高度图，改变下次雪崩流向（**这是"连环雪崩" Boss 关的机制基础**）

### 4.8 技术架构 — 数据结构（TypeScript）

```typescript
type CannonType = 'standard' | 'heavy' | 'cluster' | 'thermo' | 'ap';

// 危险区类型枚举（与 2.3 雪层状态系统对齐）

  | 'critical'   // 🔴 临界层 — 必须主动清掉
  | 'unstable'   // 🟡 不稳定层 — 受震动会松动
  | 'stable'     // 🟢 稳定层 — 安全，无需处理（一般不在 danger_zones 中列出）

  size: number;
}

// 玩家存档（云同步）
interface PlayerSave {

  coins: number;
  unlockedLevels: number;

// 关卡配置（CDN 下发）

  ammo: Record<CannonType, number>;
  stars_threshold: {
    time_3star: number;
    time_2star: number;
    ammo_3star_pct: number;
    ammo_2star_pct: number;
  };
}

// 单局结果上报
interface LevelResult {
  levelId: number;

  timestamp: number;
```

### 4.9 性能预算

| 指标 | 目标 | 备注 |
|---|---|---|
| 主包大小 | ≤ 4 MB | 微信小游戏主包限制 |

### 5.2 商业模式（占位 — 后补）

按产品负责人要求，本节为占位，列出后续可评估的变现方向，供商业团队决策：
