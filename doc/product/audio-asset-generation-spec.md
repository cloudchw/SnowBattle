## 1. 产品判断

SnowBattle 是一款明亮雪山滑雪跑酷小游戏。音频目标不是做厚重写实的滑雪模拟，而是让玩家通过声音立即理解：

1. 当前在哪个雪山场景。
2. 当前速度是否变快。
3. 收集、闪避、碰撞、失败、通关是否成功。
4. 天气是否正在影响视野和操控。
5. UI 操作是否轻快、可靠、不打断节奏。

M2 Demo 的音频验收标准：不开画面也能听出“明亮雪山、滑雪移动、金币收集、危险预警、碰撞反馈、通关结算”六类核心信息。

## 2. 全局音频方向

### 2.1 统一声音世界

- 核心风格：明亮、轻快、冒险、冰雪、跑酷、移动小游戏。
- 主题母题：上行四音动机 `1-2-5-6`，可在不同调性中移调，用短笛/钟琴/合成铃音/弦乐拨奏表达。
- 基准调性：D major / B minor 亲缘体系，冰原和暴风雪可转向 D dorian、B minor、E minor。
- 节奏核心：轻快 4/4，带滑行感的连续八分音符或十六分音符 ostinato。
- 声音质感：清澈高频、短混响、轻压缩、低频克制，避免手机扬声器糊成一团。
- 情绪曲线：初学者轻松，森林紧张，冰原晶莹，暴风雪压迫，UI 温暖可靠。
- 音频连续性：不同场景可换配器和强度，但主母题、冰雪铃音、滑行动感要保持同一品牌感。

### 2.2 交付制式

| 类型 | AI 生成源格式 | 游戏导入建议 | 声道 | 采样率/位深 | 响度目标 | 尾部 |
|---|---|---|---|---|---|---|
| BGM 循环 | WAV | OGG/MP3 | Stereo | 48kHz/24bit | -18 LUFS integrated，峰值 <= -1 dBTP | 可无缝 loop，尾部 <= 500ms |
| 环境声 Ambience | WAV | OGG | Stereo | 48kHz/24bit | -26 至 -22 LUFS | 可无缝 loop |
| UI 音效 | WAV | WAV/OGG | Mono 或 Stereo | 48kHz/24bit | -20 至 -16 LUFS | 尾部 <= 300ms |
| 游戏动作 SFX | WAV | WAV/OGG | Mono 优先 | 48kHz/24bit | -18 至 -14 LUFS | 尾部 <= 500ms |
| 危险/失败 Stinger | WAV | OGG | Stereo | 48kHz/24bit | -16 至 -14 LUFS | 尾部 <= 1200ms |
| 角色短语音占位 | WAV | OGG | Mono | 48kHz/24bit | -18 LUFS | 尾部 <= 300ms |

运行时建议：

- BGM 用 2-3 个强度层：`base`、`intensity`、`danger`。
- 天气用可叠加环境层：晴天弱风、小雪、暴风雪、雾、夜间、狂风。
- 高频 UI 音效控制在 1kHz-6kHz，不要刺耳。
- 金币和道具音效保持短、亮、可连击。
- 碰撞音效要喜剧化，避免暴力和惊吓。
- AI 生成源建议保留 WAV 母版；进入 Cocos/小游戏包体前再压缩为 MP3/OGG。BGM 可用 MP3 96kbps 或 OGG 96-128kbps；短 SFX 可用 OGG/MP3 128kbps；需要极低延迟的 UI/SFX 可保留 WAV。
- 当前工程音频接入可先覆盖 `jump`、`dive`、`coin`、`collision`、`powerUp`、`backgroundMusic` 六类基础 key，再扩展到本文档的完整命名。

### 2.3 通用正向加分词


## 3. 编号规则

编号格式：

```text
SB-AUD-<阶段>-<场景编号>-<序号>-<类型>-<名称>
```

示例：

```text
SB-AUD-M2-S01-001-BGM-BEGINNER-LOOP
SB-AUD-M2-S01-008-SFX-COIN-PICKUP
SB-AUD-M3-S04-004-AMB-BLIZZARD-LOOP
```

类型说明：

| 类型 | 含义 |
|---|---|
| `BGM` | 背景音乐，可循环 |
| `AMB` | 环境声，可叠加循环 |
| `SFX` | 游戏动作、收集、碰撞、道具音效 |
| `UI` | 菜单、按钮、HUD、结算反馈 |
| `STINGER` | 短音乐提示，如危险、失败、通关 |
| `VOICE` | 角色短语音或拟声，占位阶段可不用 |

## 4. 场景包连续性规则

同一场景包内的多个音频必须满足：

- 调性相邻：BGM、stinger、UI 音尽量使用同一主音或五度关系。
- 节奏相容：动作音效落点不能和 BGM 主拍冲突，金币连击可贴 1/8 或 1/16 网格。
- 音色统一：同场景共用冰雪铃音、短木琴、轻电子鼓、雪粉 Foley。
- 混响一致：户外雪地短混响，森林略有树间反射，冰原更明亮，暴风雪更宽但不能糊。
- 动态分层：环境声最低，BGM 中等，关键反馈最高，UI 清晰但不刺耳。
- 情绪接续：菜单 -> 进入关卡 -> 滑行 -> 危险 -> 失败/通关，不能像来自不同游戏。

如果生成工具支持参考音频，建议先生成每个场景包的 `001-BGM`，后续同场景音频都引用该 BGM 作为风格锚点。

### SB-AUD-M2-S00-001-BGM-BOOT-LOGO：启动 Logo 音乐

用途场景：Boot/Logo、首次打开游戏或启动页品牌提示。

规格：

- 时长制式：6-8 秒，Stereo，48kHz/24bit WAV 母版，运行时可压缩 MP3/OGG，-18 LUFS。
- 乐理硬性参数：D major，4/4，80 BPM，2 小节上扬 + 1 小节收束。
- 调式/和声/旋律特征：使用主母题 `1-2-5-6` 的慢速晶莹版本，最终落到 I 和弦。
- 配器音色：冰晶钟琴、柔和弦乐 pad、轻微雪风、极轻木琴点缀。
- 曲风氛围：清澈、欢迎、品牌感、轻冒险。
- 情绪画面适配：雪山 Logo 或启动页淡入，接入主菜单不突兀。
- 动态结构：0-2s 冰晶出现，2-5s 主题上扬，5-8s 柔和收束。
- 正向加分词：snow crystal logo、cute arcade、soft sparkle、冰雪清澈、品牌开场。
- 方向排除词：不要厚重电影片头，不要黑暗，不要人声歌词，不要史诗合唱。

### SB-AUD-M2-S01-001-BGM-BEGINNER-LOOP：初学者晴天雪道 BGM

用途场景：第 1-5 关基础滑行、M2 Demo 主循环。

规格：

- 时长制式：45 秒，可无缝循环，Stereo，48kHz/24bit WAV，-18 LUFS，峰值 <= -1 dBTP。
- 乐理硬性参数：D major，4/4，128 BPM，8 小节 intro + 24 小节 loop 主体，结尾回到 I 和弦。
- 调式/和声/旋律特征：I-V-vi-IV 为主，副歌感不要太强；主母题 `1-2-5-6` 用钟琴和短笛轻轻出现；旋律跨度不超过 1 个八度。
- 配器音色：钟琴、木琴、弦乐拨奏、轻电子鼓、软合成 pad、少量雪粉上行 riser。
- 曲风氛围：清爽、轻快、冒险、亲和，移动端休闲跑酷。
- 情绪画面适配：适配明亮雪道、金币路线、新手跳跃和左右闪避。
- 动态结构：0-8s 建立雪山主题，8-32s 加入鼓组和 ostinato，32-45s 回落并无缝回到开头。
- 正向加分词：冰雪清澈、轻快滑行、Q版冒险、无缝循环、手机扬声器清晰。
- 方向排除词：不要歌词，不要摇滚吉他主导，不要电影史诗，不要圣诞歌，不要悲伤，不要复杂爵士和弦。

### SB-AUD-M2-S01-002-AMB-CLEAR-SNOWFIELD：晴天雪地环境声

用途场景：晴天雪道底层环境循环。

规格：

- 时长制式：30 秒无缝循环，Stereo，48kHz/24bit WAV，-26 LUFS。

- 配器音色：轻风、细雪粒、远处雪坡空气、非常弱的滑雪场空间反射。
- 曲风氛围：安静、清澈、开阔。
- 情绪画面适配：不抢 BGM，给雪山空间感。
- 动态结构：稳定循环，10 秒一次极轻微风向变化。
- 正向加分词：清爽空气、细雪、远山空间、低噪点。
- 方向排除词：不要暴风雪，不要鸟叫主导，不要人群，不要水流，不要强风啸叫。

### SB-AUD-M2-S01-004-SFX-SKI-TURN：左右转向扫雪

用途场景：右屏左滑/右滑转向反馈。

规格：

- 时长制式：0.25-0.45 秒，Mono，48kHz/24bit WAV，-17 LUFS。
- 乐理硬性参数：非定音，可有上扬 2-3 半音的短噪声 sweep。
- 调式/和声/旋律特征：无和声，短促扫雪瞬态。
- 配器音色：雪板侧刃刮雪、粉雪向外甩开。

## 17. 生成与验收流程

### 17.1 推荐生成顺序

1. 先生成 `SB-AUD-M2-S01-001-BGM-BEGINNER-LOOP`，锁定品牌音乐语言。
2. 生成 S01 环境声和玩家滑行基础音效。
3. 生成金币、道具、警告、危险、碰撞四类高频反馈。
4. 生成主菜单和结算 UI 音频。
5. 扩展森林、冰原、暴风雪章节 BGM 和环境声。
6. 最后生成角色短拟声和高阶道具组合音。

### 17.2 单段音频验收标准

- BGM loop 接缝听不出跳变。
- 高频反馈在手机扬声器上仍可分辨。
- 金币、护盾、加速、磁铁四类奖励音色不同。
- 警告和危险有明确层级：警告提醒准备，危险要求立即反应。
- 碰撞失败喜剧化，不恐怖、不暴力。
- 环境声不会遮挡金币、警告、UI 反馈。
- 同一场景包内调性、混响、音色不冲突。
- 不包含歌词、人声水印、版权旋律或可识别商业音乐片段。

### 17.3 M2 Demo 音频验收标准

- 主菜单有温暖冰雪氛围。
- 进入关卡时能听出“开始滑行”。
- 持续滑行时有轻雪摩擦。
- 跳跃、俯冲、落地有即时反馈。
- 金币、护盾、加速、磁铁能用声音区分。
- 警告和危险不会混淆。
- 碰撞和失败有轻喜剧反馈。
- 通关结算有清晰奖励感。
