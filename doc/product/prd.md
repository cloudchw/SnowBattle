## 1. 产品定位

### 1.1 基本信息

| 维度 | 内容 |
|---|---|
| **产品名（暂定）** | 《滑雪大冒险》（Snow Battle） |
| **一句话定位** | 一款"在雪山上极限滑雪、躲避障碍、挑战自我"的 **2.5D 等距视角**跑酷小游戏 |
| **目标平台** | 微信小游戏 + 抖音小游戏（手机端） |
| **核心情绪** | 刺激 + 紧张 + 爽快（速度感 + 精准操控的成就感） |
| **目标用户** | 微信生态内 12–35 岁休闲玩家；喜欢《滑雪大冒险》《神庙逃亡》类跑酷 |
| **典型场景** | 通勤/排队/午休的 1–3 分钟碎片时间 |
| **差异化卖点** | **2.5D 等距 + 手势操控 + 深度策略障碍系统** + 团结引擎 3D 渲染（立体模型/光照/粒子） |

> **为什么 2.5D 而非真 3D**：2.5D 等距视角保留"读地形"的策略性（玩家能纵览障碍布局），操控更精准（手势而非重力惯性），性能与工期可控。团结引擎的 3D 能力让 2.5D 画面比纯 2D 更立体（真实模型、动态光照、粒子雪），但规避了真 3D 的相机/物理/性能复杂度。

### 1.2 故事背景

阿尔卑斯山区年轻滑雪爱好者，从雪山之巅挑战极限雪道，躲避悬崖、树林、暴风雪与其他滑雪者。

### 1.3 核心循环

（与 v2.1 一致）

**单局循环（60–180 秒）**：开局→选角色→进入雪道→自动下滑+手势操控→躲避障碍+收集→✅过关/❌失败。

**三大张力**：速度vs控制、风险vs奖励、时机vs预判。

## 2. 核心玩法机制

### 2.1 战场布局（2.5D 等距，**团结引擎 3D 渲染**）

```
┌──────────────────────────────────────┐
│  顶部 HUD：⏱时间 🪙金币 ⭐分数 [道具] │
├──────────────────────────────────────┤
│        ❄️ 雪山远景（3D 远景层）       │
│       ╱╲  🌲🌲🌲  ← 3D 树林模型      │
│      ╱  ╲                             │
│     ╱    ╲______  ← 缓坡（高度图）   │
│    ╱  🎿→   🧑‍🦳    ← 玩家+NPC        │
│   ╱              ╲                    │
│  🏁起点          🏁终点              │
│  （等距相机固定角度俯视，3D 模型+光照）│
└──────────────────────────────────────┘
```

> **团结引擎实现 2.5D**：场景用 3D 模型（角色/树/岩石）+ 等距相机（正交或固定角度透视），配合 URP 光照与粒子。比 Cocos 的 2D 贴图更立体，但视角固定、操控简单（手势）。

### 2.2 手势操控系统（**保留 v2.1，不变**）

| 动作 | 操作 | 效果 | 冷却 |
|---|---|---|---|
| 跳跃 | 左屏上滑 | 跳跃，躲地面/低矮障碍 | 0.8s |
| 俯冲 | 左屏下滑 | 快速下落，躲空中障碍/加速 | 0.6s |
| 左移 | 右屏左滑 | 向左调整方向 | 无 |
| 右移 | 右屏右滑 | 向右调整方向 | 无 |
| 急停 | 右屏下滑 | 短暂减速 | 2s |
| 加速 | 右屏上滑 | 短暂冲刺 | 3s |

**设计：手势操控，无虚拟按钮**（沉浸感）。

### 2.3 障碍物系统（**保留 v2.1 完整设计**）

8 种障碍（树/悬崖/山坡/其他滑雪者/雪堆/冰面/温泉/落石）+ 三态（安全/警告/危险）+ 隐藏信息（冰裂/雪崩预警/暗冰）。**3D 模型化**（团结引擎渲染立体障碍 + 危险态发光描边）。

### 2.4 天气系统（**保留 v2.1 六种**）

晴天/小雪/暴风雪/大雾/夜间/狂风。**团结引擎 3D 粒子 + 后处理**（粒子雪、体积雾、夜间点光源）。

### 2.5 道具系统（**保留 v2.1 八种 + 组合**）

金币/护盾/加速器/磁铁/冰镐/降落伞/照明弹/雪铲 + 组合效果。3D 模型化。

### 2.6 角色系统（**保留 v2.1 六角色**）

小明/雪豹/冰精灵/暴风骑士/极地探险家/滑雪大师。**3D 模型 + 动画**（滑行/跳跃/俯冲/撞击/胜利）。

## 4. 美术与技术架构

### 4.1 引擎选型（**变更**）

| 维度 | v2.1（Cocos） | **v3.1（团结引擎）** |
|---|---|---|
| 引擎 | Cocos Creator 3.8.8 | **团结引擎 Tuanjie Engine** |
| 语言 | TypeScript | **C#** |
| 视角 | 2.5D 等距 | **2.5D 等距（不变）** |
| 渲染 | 2D 贴图 | **URP + 3D 模型 + 等距相机** |
| 物理 | 自研网格 | **团结引擎物理（2.5D 简化：3D Collider + 固定视角）** |
| 小游戏支持 | 微信首推 | **团结引擎原生支持微信/抖音导出** |

**为什么团结引擎做 2.5D**：
- ✅ 3D 渲染让 2.5D 画面更立体（模型/光照/粒子，优于 Cocos 2D 贴图）
- ✅ Unity 工具链成熟（编辑器、Prefab、动画、物理）
- ✅ 国内小游戏原生支持
- ❌ 取舍：包体比 Cocos 略大，需压缩

### 4.2 物理方案（**变更**）

| 维度 | v2.1 | **v3.1** |
|---|---|---|
| 方案 | 自研网格+触发器 | **团结引擎物理（简化）** |
| 玩家移动 | 网格导航 | ** lanes 车道 + 3D 位置插值**（2.5D 等距下离散车道） |
| 碰撞 | 网格位置 | **3D Collider + Trigger**（等距视角下的碰撞） |
| 地形 | 高度图叠加 | **高度图（影响速度视觉，不做真物理坡度）** |

> **2.5D 物理简化**：不像真 3D 那样用完整 PhysX 刚体滑雪惯性，而是保留 v2.1 的"车道+触发器"思路（用团结的 Collider 实现），降低复杂度。地形高度主要做视觉表现（坡度加速感通过脚本模拟，非真物理）。

### 4.3 后端 + 数据流（**不变**）

微信云开发 CloudBase：云函数/云数据库/云存储 CDN。关卡热更：主包 1-3 章，4 章后 CDN。

### 4.4 数据结构（**TS → C#，坐标保持 [x,y]**）

```csharp
public enum ObstacleType { Tree, Cliff, Hill, Skier, SnowPile, Ice, HotSpring, Rock }
public enum WeatherType { Clear, LightSnow, Blizzard, Fog, Night, Wind }

[System.Serializable]
public class Obstacle {
    public ObstacleType type;
    public float[] position;   // [x, y] 2.5D 平面，等距映射到 3D
    public string state;
}

[System.Serializable]
public class LevelConfig {
    public string id, name, terrain;
    public WeatherType weather;
    public List<Obstacle> obstacles;
    public CollectibleConfig collectibles;
    public Goals goals;
    public StarThreshold stars_threshold;
}
```

> **关卡 JSON 与 v2.1 完全兼容**，50 关数据直接复用。

### 4.5 性能预算（**2.5D 标定**）

| 指标 | v2.1 | **v3.1（2.5D/团结）** | 说明 |
|---|---|---|---|
| 主包 | ≤4MB | **≤4MB** | 团结引擎需裁剪+压缩 |
| 总包 | ≤20MB | **≤20-25MB** | 3D 模型略增，分包控制 |
| 帧率 | 60FPS | **60FPS**（低端30） | 2.5D 渲染开销适中 |
| 内存 | ≤300MB | **≤300MB** | |
| 同屏障碍 | 30/20/10 | **25/15/8** | 3D 模型分级 |
| Draw Call | — | **≤80** | URP 合批 |

### 4.6 美术资产（**2.5D / 3D 模型**）

| 类别 | v2.1（2D） | **v3.1（2.5D 3D 模型）** |
|---|---|---|
| 角色 | 立绘+表情 | **3D 模型 + 骨骼动画**（6 角色，低面） |
| 场景 | 2D prefab | **3D 等距场景**（6 章节） |
| 障碍 | 2D 图 | **3D 模型**（低面，LOD） |
| 特效 | 序列帧 | **粒子系统** |
| UI | 2D | UGUI（叠 2.5D 上） |

工作量：比 v2.1（纯 2D）增（3D 模型），比真 3D 减（等距固定、低面）。团队：1 主美 + 1 3D 建模 × 3.5-4 月。
