## 1. 产品定位

### 1.1 基本信息

| 维度 | 内容 |
|---|---|
| **产品名（暂定）** | 《滑雪大冒险》（Snow Battle） |
| **一句话定位** | 一款"在雪山上极限滑雪、躲避障碍、挑战自我"的2.5D跑酷小游戏 |
| **目标平台** | 微信小游戏 + 抖音小游戏（手机端首发，不考虑PC/Web） |
| **核心情绪** | 刺激 + 紧张 + 爽快（速度感 + 精准操控的成就感） |
| **目标用户** | 微信生态内 12–35 岁休闲玩家；喜欢《滑雪大冒险》《神庙逃亡》类跑酷游戏 |
| **典型场景** | 通勤/排队/午休的单次 1–3 分钟碎片时间 |
| **差异化卖点** | "2.5D等距视角 + 手势操控 + 深度策略障碍系统" — 区别于传统2D横版跑酷的沉浸感和策略深度 |

### 1.2 故事背景

> 你是阿尔卑斯山区一名年轻的滑雪爱好者。每个冬天，你都会挑战最危险的雪道 — 从雪山之巅一路滑到山谷。但这些雪道并不安全：陡峭的悬崖、密集的树林、不可预测的暴风雪，还有其他滑雪者突然出现。你需要凭借精湛的技巧和冷静的判断，在速度与安全之间找到平衡，完成一次又一次的极限挑战。

### 1.3 核心循环

**单局循环（无尽模式，60–180 秒）**

```
开局 → 选择角色/装备 → 进入雪道
     → 自动向前滑行（速度随地形变化）
     → 手势操控：左屏上滑跳跃/下滑俯冲，右屏滑动调整方向
     → 躲避障碍物 + 收集金币/道具
     → 三种结果：
         ✅ 完成关卡目标 = 过关（按用时/收集/技巧评 1-3 星）
         ❌ 撞击障碍物 = 失败（复活/重试）
         ❌ 掉落悬崖 = 失败（直接结束）
```

**单关循环（关卡模式，60–120 秒）**

```
开局 → 查看关卡目标（到达终点/收集X金币/存活X秒）
     → 进入预设地形
     → 手势操控完成目标
     → 三种结果：
         ✅ 达成目标 = 过关（星级评定）
         ❌ 失败 = 重试/复活
```

**跨局循环**

```
通关 → 获得金币 + 经验 → 升级装备/解锁角色 → 挑战下一关
失败 → 激励视频复活一次 / 重试
```

### 1.4 三大核心张力

设计之魂 — 每个机制都应至少强化其中之一：

1. **速度 vs 控制**：速度快得分高，但反应时间缩短 — 贪快会送命
2. **风险 vs 奖励**：危险路径（悬崖边、树林间）金币更多，但难度倍增 — 冒险可能一无所有
3. **时机 vs 预判**：跳跃和俯冲有冷却时间，必须预判障碍物出现时机 — 乱按会送命

### 2.2 手势操控系统（核心 UX）

| 动作 | 操作 | 效果 | 冷却 |
|---|---|---|---|
| **跳跃** | 左屏上滑 | 向上跳跃，躲避地面障碍和低矮障碍 | 0.8s |
| **俯冲** | 左屏下滑 | 向下俯冲，快速下落，躲避空中障碍或加速 | 0.6s |
| **左移** | 右屏左滑 | 向左调整方向（相对2.5D视角） | 无 |
| **右移** | 右屏右滑 | 向右调整方向 | 无 |
| **急停** | 右屏下滑 | 短暂减速，用于精确避障 | 2s |
| **加速** | 右屏上滑 | 短暂加速，用于冲刺 | 3s |
| **缩放** | 双指捏合 | 缩放视角（用于观察远处地形） | 无 |
| **旋转** | 双指旋转 | 旋转视角（2.5D允许±30°旋转） | 无 |

**关键设计决策：手势操控，无虚拟按钮。** 玩家通过直觉手势控制，提升沉浸感和操控爽快度。

**新手过渡（第 1-3 关）**：
- 仅开放跳跃和左右移动
- 第 4 关引入俯冲
- 第 7 关引入急停和加速
- 障碍物间距大，容错率高

### 2.3 障碍物系统（深度策略维度）

#### 2.3.1 障碍物类型

| # | 名称 | 视觉 | 机制 | 应对方式 | 危险等级 |
|---|---|---|---|---|---|
| 1 | **树木** | 🌲 绿色针叶 | 静态障碍，密集分布 | 左右闪避或跳跃越过低矮枝 | ⭐⭐ |
| 2 | **悬崖** | ⬛ 深色断崖 | 必须跳跃通过，否则掉落 | 精准跳跃（时机很重要） | ⭐⭐⭐ |
| 3 | **山坡** | ⬛ 陡峭斜面 | 上坡减速，下坡加速 | 利用加速/控制方向 | ⭐ |
| 4 | **其他滑雪者** | 🧑‍🦳 动态人物 | 随机移动，不可预测 | 预判路径，提前变向 | ⭐⭐ |
| 5 | **雪堆** | ⬜ 白色堆积 | 可撞碎（减速）或跳过 | 根据速度选择策略 | ⭐ |
| 6 | **冰面** | 🔵 蓝色透明 | 加速但失控，难以转向 | 保持直线，提前变向 | ⭐⭐ |
| 7 | **温泉** | ♨️ 蒸汽上升 | 减速区域，影响节奏 | 快速通过或绕行 | ⭐ |
| 8 | **落石** | 🪨 棕色岩石 | 随机出现，需紧急反应 | 急停或跳跃 | ⭐⭐⭐ |

#### 2.3.2 障碍物状态系统

障碍物有 3 种可视化状态（玩家"观察"的核心信息）：

| 状态 | 颜色 | 含义 | 玩家应对 |
|---|---|---|---|
| 🟢 **安全** | 正常颜色 | 无威胁，正常通过 | 正常操控 |
| 🟡 **警告** | 黄色脉动 | 即将变化，需提前准备 | 提前变向或准备跳跃 |
| 🔴 **危险** | 红色强烈脉动 | 随时可能触发，必须立即应对 | 紧急跳跃/俯冲/急停 |

**隐藏信息（高阶钻研点）**：
- **冰裂**（蓝色裂纹）：冰面即将破裂，必须快速通过
- **雪崩预警**（山顶积雪松动）：即将引发雪崩，必须加速逃离
- **暗冰**（冰面下隐藏的冰层）：表面正常但极滑，需预判

### 2.4 天气系统（全关通用）

| 天气 | 效果 | 视觉 | 策略影响 |
|---|---|---|---|
| **晴天** | 正常 | 明亮，视野清晰 | 基础难度 |
| **小雪** | 轻微减速 | 雪花飘落 | 视野略受影响 |
| **暴风雪** | 大幅减速 + 低能见度 | 暴风雪特效 | 难以预判，需依赖记忆 |
| **大雾** | 极低能见度 | 浓雾遮挡 | 只能看到近处障碍 |
| **夜间** | 低能见度 + 路灯照明 | 暗色调 + 路灯光晕 | 需沿着灯光路线走 |
| **狂风** | 方向偏移 | 风向旗帜飘动 | 滑行方向受影响，需提前修正 |

**天气切换机制**：天气在关卡中动态变化（如前半段晴天，后半段暴风雪），增加不可预测性。

### 2.5 道具系统（策略组合维度）

#### 2.5.1 可收集道具

| # | 名称 | 视觉 | 机制 | 持续时间 | 稀有度 |
|---|---|---|---|---|---|
| 1 | **金币** | 🪙 金色圆形 | 收集用于购买/升级 | 永久 | 普通 |

| 5 | **冰镐** | 🪓 银色工具 | 冰面上不失控 | 10秒 | 史诗 |

### 2.7 计分与失败判定

#### 2.7.1 关卡模式过关条件

**过关条件（同时满足）**：
1. 到达终点（或完成指定距离）
2. 收集 ≥ 目标数量的金币（如有）
3. 存活时间 ≥ 目标时间（如有）
4. 没有掉落悬崖

#### 2.7.2 星级评定

| 星级 | 条件 | 策略深度 |
|---|---|---|
| ⭐ | 完成关卡基本目标 | 通过即可 |
| ⭐⭐ | 用时 ≤ 目标70% + 金币 ≥ 目标120% | 需要速度+收集 |
| ⭐⭐⭐ | 用时 ≤ 目标50% + 金币 ≥ 目标150% + 0次撞击 | 需要完美操控 |

追求 3 星 = 高度策略性，是核心玩家复玩动力。

#### 2.7.3 失败场景（带喜剧动画）

- 撞击树木 → "哎呦！这树怎么长在这" + 重试/复活
- 掉落悬崖 → "啊——" + 慢动作坠落 + 重试/复活
- 被其他滑雪者撞飞 → "兄弟你赶着投胎吗" + 重试/复活
- 被雪崩掩埋 → "这雪崩来得真不是时候" + 重试/复活

### 2.8 无尽模式

#### 2.8.1 模式规则

- 解锁条件：通关第 1 章（第 5 关）
- 自动向前滑行，速度逐渐加快
- 障碍物密度和复杂度随距离增加
- 天气随机变化
- 最终以撞击/掉落结束

#### 2.8.2 无尽模式特殊机制

| 机制 | 效果 | 触发条件 |
|---|---|---|
| **连击系统** | 连续躲避障碍获得连击加分 | 连续躲避5次以上 |
| **完美闪避** | 极限距离躲避获得额外加分 | 距离障碍<10px时躲避 |
| **速度挑战** | 速度越快得分倍率越高 | 持续高速滑行 |
| **金币雨** | 大量金币集中出现 | 随机事件 |

#### 2.8.3 排行榜

- 本地排行榜（历史最高分）
- 微信好友排行榜
- 全球排行榜
- 每日挑战（固定种子，公平竞争）

## 3. 关卡设计与难度曲线

### 3.1 关卡总量与章节划分

主线 **50 关**，预计玩家 7-14 天通关（休闲节奏）。**周更关卡**从第 4 周起每周追加 3 关，无限延展。

| 章节 | 关卡范围 | 主题 | 新机制引入 |
|---|---|---|---|
| **第 1 章：初学者** | 1-5 | 基础操作 | 跳跃、左右移动，无危险障碍 |
| | 6-10 | 简单障碍 | 树木、雪堆，引入俯冲 |
| **第 2 章：森林** | 11-15 | 密集树林 | 树木障碍密集，需要精准闪避 |
| | 16-20 | 悬崖初现 | 悬崖障碍，需要跳跃通过 |
| **第 3 章：高山** | 21-25 | 地形变化 | 山坡加速/减速，高度差 |
| | 26-30 | 天气系统 | 引入天气变化（小雪、大雾） |
| **第 4 章：冰原** | 31-35 | 冰面挑战 | 冰面障碍，失控机制 |
| | 36-40 | 暴风雪 | 暴风雪天气，低能见度 |
| **第 5 章：极限** | 41-45 | 综合挑战 | 全机制混合，高密度障碍 |
| | 46-50 | 终极考验 | Boss关，极端天气+复杂地形+多障碍 |

### 3.3 难度曲线

| 关卡段 | 平均障碍密度 | 天气复杂度 | 道具可用性 | 操控要求 |
|---|---|---|---|---|
| 1-5 | 低 | 无 | 无 | 基础跳跃+移动 |
| 6-10 | 低-中 | 无 | 护盾/加速器 | 引入俯冲 |
| 11-20 | 中 | 小雪 | 全道具 | 精准闪避 |
| 21-30 | 中-高 | 小雪/大雾 | 全道具 | 地形适应 |
| 31-40 | 高 | 暴风雪/夜间 | 全道具 | 极端条件操控 |
| 41-50 | 极高 | 全天气 | 全道具+组合 | 完美操控 |

**关键设计**：操控要求逐步上升 — 早期"按跳跃就能过"，后期"必须预判3步以上"，强化策略感和压力感。

**第 1-5 关无危险障碍的设计意图**：刻意分离"手势操控训练"与"障碍物应对训练"。1-3 关专注跳跃+移动肌肉记忆；4-5 关引入俯冲；从第 6 关起才叠加危险障碍，避免新手同时面对多个变量。这是教学排序逻辑，并非障碍系统非 MVP 必需 — 障碍系统从第 6 关起是核心循环不可分割的一部分。

### 3.4 Boss 关设计（46-50 关）

- **第 46 关：暴风雪山** — 暴风雪天气 + 密集树林 + 悬崖，能见度极低
- **第 47 关：冰原迷宫** — 大面积冰面 + 隐藏暗冰 + 其他滑雪者干扰
- **第 48 关：雪崩追逐** — 身后雪崩追赶，必须保持速度同时躲避障碍
- **第 49 关：终极山路** — 全障碍类型混合 + 极端天气切换
- **第 50 关：雪山之巅** — 8分钟超长关卡，全机制综合考验，终点在雪山顶

### 3.5 周更关卡设计原则

- 每周 3 关，主题化（如"圣诞特辑"、"新年挑战"）
- 难度对标主线第 30-40 关（中高难度，给老玩家挑战）
- 含 1 关"创意关"：引入限时机制（如限时 30s 速攻、限跳跃次数极限挑战）

## 4. 美术与技术架构

### 4.1 美术 — 角色设计

- 表情库：滑行（专注）/ 跳跃（兴奋）/ 俯冲（紧张）/ 撞击（痛苦）/ 胜利（开心）

**可解锁角色（5 种造型）**：
1. **雪豹**：银白色毛皮装，动物耳朵装饰
2. **冰精灵**：透明蓝色斗篷，发光效果
3. **暴风骑士**：深蓝色铠甲，披风飘动
4. **极地探险家**：厚重冬装，探险帽，背包
5. **滑雪大师**：专业竞赛服，号码牌

每个角色被撞击时有专属的滑稽动画（如冰精灵："我的斗篷！"）

### 4.5 技术架构 — 引擎选型

| 候选 | 评估 | 决策 |
|---|---|---|
| **Cocos Creator 3.8.x** | 微信小游戏官方首推、2.5D 友好、TS 强类型、生态成熟 | ✅ **选定** |
| Laya | 性能略好但社区小、TS 体验弱 | ❌ |
| PixiJS + 自研框架 | 灵活但工程量大、缺编辑器 | ❌ |
| Egret | 老牌但更新缓慢 | ❌ |

### 4.6 技术架构 — 物理方案

**关键决策：自研轻量物理模拟，不用真物理引擎。**

| 维度 | 真物理（Cannon.js/Bullet） | 自研轻量 | 决策 |
|---|---|---|---|
| 表现可控 | 难调，物理行为不可预测 | 流向可艺术化 | 自研胜 |
| 性能 | 大量刚体昂贵 | 粒子系统 + 高度图便宜 | 自研胜 |
| 开发量 | 接入快，调参慢 | 2-3 周自研 | 真物理快但后续痛苦 |
| 地形交互 | 难做 | 高度图天然支持 | 自研胜 |

**自研方案核心**：
- 玩家移动 = **网格导航系统**（Grid 32×32 或 64×64），玩家根据手势输入在网格上移动
- 障碍物 = **触发器区域**（Trigger Zone），碰撞检测基于网格位置
- 地形效果 = **高度图叠加**，影响玩家速度和方向
- 天气效果 = **后处理 shader**，影响视觉和操控

### 5.2 商业模式（占位 — 后补）
